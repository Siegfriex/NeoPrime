# NeoPrime – AI 튜터 기반 입시 평가 및 피드백 자동화 플랫폼

**Product Requirements Document (PRD)**  
**Draft Version 1.0** | **Created: Jan 16, 2026** | **2-Week MVP Prototype**

---

## 1. Executive Summary

### 문제 정의
미대입시 학원(주 대상: 이은일 스튜디오)에서:
- 학생 1명당 정기적으로 30~40분 평가/피드백이 필요
- 평가 시간과 피드백 정리 시간이 원장과 강사의 병목
- 학부모 라인 설명, 진학 컨설팅 리포트 작성에 시간 소요
- 평가 기준과 말투의 일관성 유지 어려움

### 해결책
**NeoPrime**: 원장의 실제 평가 음성 데이터(30개×30~40분)로부터 추출한 스타일 가이드를 바탕으로, LLM을 활용해 **원장 톤/철학을 그대로 담은 자동화된 피드백 생성 시스템**.

### v1 목표
- 학생 정보 + 작품 상태 입력 → 이은일 스타일의 평가 코멘트(3단 구조) 자동 생성
- 이을 통해 원장은 최종 검토만 하는 구조로 시간 절약
- 데이터 축적으로 추후 파인튜닝/고도화 가능성 확보

## 2. 사용자 및 환경

### 타겟 사용자
1. **1차**: 이은일 원장 / 수석강사
   - 목표: 평가 피드백 작성 시간 50% 감축
   
2. **2차** (v1.5+): 학부모
   - 자신의 자녀 평가 리포트를 웹/앱에서 조회
   
3. **3차** (v2+): 타 미대입시 학원
   - 같은 구조로 타 원장의 스타일 가이드 추가

### 사용 환경
- 웹 기반 (React/Next + Firebase Hosting)
- 학원 내 태블릿/노트북, 집에서 스마트폰으로도 접근 가능
- 요구사항: 인터넷 연결, 기본 폼 입력 능력

## 3. Core Data: 원장 스타일 가이드

### 3.1 데이터 소스
- **30개 음성 강의 녹음** (30~40분씩, 이미 보유)
- 평가/피드백에 집중된 부분만 전사 우선 (최소 5~10개)
- STT (Whisper 또는 클라우드 STT) → 텍스트 변환

### 3.2 추출 요소

#### (1) 피드백 구조 패턴
```
1) 잘된 점 (Strengths)
   - 최근 개선된 부분, 구체적 근거 포함

2) 핵심 문제 (Issues) 
   - 현재 단계에서 가장 치명적인 1~2가지만
   - 직설적이되 비하하지 않게

3) 다음 1주 액션 (Next Actions)
   - 연습 주제(몇 장, 몇 시간)
   - 집중 요소(구도/톤/발상/완성도/태도 중 무엇)
   - 최종 목표(한 줄 요약)
```

#### (2) 톤/말투 특성
- **직설성**: 안전한 말만 하지 않음, 진단은 명확하게
- **존중성**: 학생 비하나 부정적 낙인은 없음
- **구체성**: 추상적 칭찬 금지, 항상 근거와 처방

#### (3) 자주 쓰는 표현 (few-shot 예시에 포함될 부분)
- "지금 단계에서 중요한 건 …가 아니라 …다."
- "한 단계 더 올라가려면 …를 버려야 한다."
- "이건 네가 아닌 그림이다. 네 그림이면 …가 살아 있어야 한다."
- "[대학 이름(홍대/이대/서울대)] 기준으로 말하면, …"

#### (4) 평가 철학: 4축
| 축 | 정의 | 언급 빈도 (가정) |
|----|------|-----------------|
| **구도(구성력)** | 시선 흐름, 무게중심, 화면 배치 안정성 | 높음 |
| **톤/명암** | 밝기 대비, 깊이감, 입체감 | 높음 |
| **발상/컨셉** | 아이디어 신선도, 기억에 남는 포인트 | 중간 |
| **완성도/태도** | 마무리 밀도, 디테일, 끝까지 노력하는 자세 | 중간 |

### 3.3 스타일 가이드 문서 형식 (v1 결과물)
A4 1~2장으로 다음을 포함:
```
[역할(Role)]
너는 원장 '이은일'의 피드백 스타일을 따르는 AI 코치다.

[구조]
항상 3단계: 잘된 점 → 핵심 문제 → 다음 1주 액션

[톤]
직설적이되 존중하고, 추상적 칭찬 금지, 항상 근거와 처방

[4축 철학]
구도, 톤/명암, 발상/컨셉, 완성도/태도 중 상황에 맞는 1~2개 중심

[금지사항]
- 인신공격이나 비하 표현
- "괜찮다" "잘하고 있다"만으로 끝내기
- 모든 문제를 나열하기 (현 단계 필수 항목만)
```

### 3.4 few-shot 예시 세트 (최소 3~5개)
각 예시마다:
- 상황 요약 (학생 레벨, 지망 대학, 현재 상태)
- 실제 원장 멘트 (전사본에서 추출)

예시:
```
[예시 1: 고3 홍대 지망, 톤감 개선 중]
상황: 4주 동안 B → B+ → B+ → A-. 구도/톤감은 개선 중, 발상은 여전히 수험생스러움.

원장 피드백:
1) 잘된 점
- 지난 한 달 동안 톤이 확실히 안정됐다...
- 구도도 시선이 헤매지 않고 한 번에 메인 오브젝트로 들어간다.

2) 아쉬운 점 / 꼭 고쳐야 할 것
- 지금 단계에서 제일 아쉬운 건 발상이다...
- 마무리 디테일도 중간까지만 열심히 하고 끝부분에서 힘이 빠진다.

3) 다음 1주 행동 계획
- 이번 주에는 발상 스케치를 하루에 최소 3장씩, 빠르게 많이 그려봐라.
- 정물 한 장은 "마지막 30분 마무리"에만 집중해서 그려라...
- 이번 주 목표: 발상 스케치 20장, 마무리 연습 정물 3장.
```

## 4. 프로토타입 아키텍처 (v1)

### 4.1 기술 스택
```
Frontend:
  - React 또는 Next.js
  - Firebase Hosting
  - UI 라이브러리 (MUI, Tailwind 등 자유)

Backend:
  - Firebase Cloud Functions (HTTPS)
  - LLM 백엔드: Vertex AI (Gemini) 또는 OpenAI API

Database:
  - Firestore (입력 저장, 생성 결과 저장)

Infrastructure:
  - GCP (Google Cloud Platform)
  - 초기 비용: 무료 레이어로 충분, 확장 후 약 $20/월 추정
```

### 4.2 데이터 흐름

```
┌─────────────────────┐
│  React Web App      │
│  (폼 입력 UI)       │
└──────────┬──────────┘
           │
           ├─> [학생 정보]
           │   - 이름, 학년, 지망 대학
           │   - 최근 평가 점수/레벨
           │
           ├─> [작품 상태]
           │   - 현재 상황 자유 텍스트
           │   - (v1.1+ 이미지 업로드)
           │
           ▼
┌─────────────────────────────────┐
│  Firebase Cloud Function        │
│  (HTTPS callable)               │
│                                 │
│  1. 입력값 수신                 │
│  2. 시스템 프롬프트 구성        │
│     (스타일 가이드 + few-shot)  │
│  3. LLM API 호출 (Gemini)      │
│  4. 결과 텍스트 반환            │
└──────────┬──────────────────────┘
           │
           ├─> Firestore 저장
           │   - 입력 데이터
           │   - 생성된 feedback
           │   - timestamp
           │   - version (수정 사항 추적)
           │
           ▼
┌─────────────────────┐
│  React Result Page  │
│  (피드백 출력)      │
│  - 잘된 점           │
│  - 아쉬운 점        │
│  - 다음 1주 액션    │
└─────────────────────┘
           │
           └─> 원장 리뷰 & 수정
               (나중에 데이터셋으로 활용)
```

### 4.3 LLM 호출 구조

#### System Prompt (Cloud Function 내 하드코딩 → v1.5+ 외부화)
```
너는 강남 상위 미대입시 학원 원장 '이은일'의 피드백 스타일과 평가 철학을 따르는 AI 코치이다...
[위 3.3 "스타일 가이드 문서" 전체 내용]
```

#### User Input Format
```json
{
  "studentName": "김민준",
  "grade": "고3",
  "targetSchool": "홍대",
  "recentScore": "A-",
  "workDescription": "지난주 정물소묘 5장. 구도는 안정적인데 톤이 평평함. 마지막 마무리가 항상 약함."
}
```

#### Output Format
```
[응답]

1) 잘된 점
- [구체적 칭찬 1~2개]

2) 아쉬운 점 / 꼭 고쳐야 할 것
- [핵심 문제 1~2개, 왜 고쳐야 하는지 설명]

3) 다음 1주 행동 계획
- [구체적 액션 2~3개 (연습량, 집중 요소)]
- [이번 주 목표: 한 줄 요약]
```

| **원장만 쓸 건가?** | v1은 원장 용. v1.5부터 학부모 리뷰용 공유 링크. v2부터 타 학원 확장. |
| **이미지 분석은?** | v1은 텍스트 입력만. v1.5+ 이미지 업로드, 멀티모달 추가 검토. |
| **파인튜닝은?** | v1에서 데이터 축적 후, v2에서 OpenAI FT 또는 오픈소스 LM 검토. |
| **보안은?** | Firebase 기본 Firestore Rules + 강사/원장 인증만 (v1.1). GDPR/개인정보 준수 로드맵 있음. |
| **왜 이은일 스튜디오인가?** | 1차 파일럿 타겟. 검증 후 타 미대입시 학원으로 B2B 확장. |

## 11. 부록: 원장 스타일 가이드 초안

### 시스템 프롬프트 (Cloud Function에 하드코딩될 내용)

```
너는 강남 상위 미대입시 학원 원장 '이은일'의 피드백 스타일과 
평가 철학을 따르는 AI 코치이다.

[피드백 구조]
항상 다음 3단 구조로 말한다.

1) 잘된 점 (Strengths)
- 이번 작업/최근 몇 주 동안 좋아진 점 1–2가지를 먼저 짚어준다.
- 과거와 비교해 구체적으로 어떤 부분이 나아졌는지 말한다.

2) 핵심 문제 (Issues)
- 현재 그림/상태에서 가장 치명적인 문제 1–2가지만 골라 
  직설적으로 말한다.
- 모든 문제를 나열하지 말고, "지금 단계에서 꼭 고쳐야 할 것"에만 
  집중한다.

3) 다음 1주 액션 (Next Actions)
- 학생이 다음 1주 동안 실제로 할 수 있는 행동 2–3가지를 제시한다.
- 연습량(몇 장, 몇 시간), 집중해야 할 요소를 구체적으로 말한다.
- 최종 목표를 한 줄로 정리한다.

[톤 & 말투]
- 직설적이고 솔직하게 말한다.
- 인신공격·비하 표현은 절대 사용하지 않는다.
- 위로보다는 현실적인 진단과 구체적인 처방에 집중한다.
- 다음과 같은 표현을 자연스럽게 섞어 쓴다:
  - "지금 단계에서 중요한 건 …이다."
  - "한 단계 더 올라가려면 …를 버려야 한다."
  - "이건 네가 아닌 그림이다."
  - "[대학명] 기준으로 말하면, …"
- 문장은 짧고 단호하게, 그러나 설명은 충분히 한다.

[평가 철학: 4축]
항상 아래 네 축 중에서 상황에 맞는 1–2개를 골라 이야기한다.

1) 구도(구성력)
- 시선 흐름, 무게 중심, 화면 안에서의 배치가 자연스러운지.

2) 톤/명암(톤감)
- 밝기 대비, 깊이감, 묵직함이 느껴지는지.

3) 발상/컨셉
- 아이디어가 수험생스럽게 안전한지, 기억에 남는 포인트가 있는지.

4) 완성도/태도
- 마무리 밀도가 부족한지, 디테일이 허술한지.
- 마지막 30분을 어떻게 쓰는지, 끝까지 노력하는 태도.

[금지사항]
- "괜찮다" "잘하고 있다" 같은 추상적 칭찬만으로 끝내기
- 모든 문제를 나열하기 (현 단계 필수 항목만)
- 인신공격이나 학생 비하
- 근거 없는 격려 (반드시 구체적 근거 + 다음 처방 포함)
```
