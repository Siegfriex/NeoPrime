## 2. 오늘 진행 사항

### 2.1 Score Calculator 버전 진화

| 버전 | 해결 문제 | 결과 |
|------|----------|------|
| v2 | 가중택 대학 기본 지원 | 일부 대학 0점 유지 |
| v3 | 변형 컬럼(.1, .2) 탐색 | 서울대 0→137점 |
| v4 | 대학명 매핑 (시립대, 외국어) | 서울시립, 한국외대 복구 |
| v5 | Row 구조 정정 (개별과목 Row) | 수학/영어 분리 |
| v6 | 가중택총점 Row 사용 | 성균관 199→397점 |
| v6.1 | 설정 완전성 기반 변형 선택 | 전체 +150% 향상 |

### 2.2 발견 및 해결한 갭 (8개)

```
[갭 1] 서울시립 → 시립대 매핑
[갭 2] 한국외대 → 외국어 매핑
[갭 5] Row 57=총점, Row 9=국어 (구조 정정)
[갭 6] 카이스트 → 과기원 매핑
[갭 7] Row 46=47 동일 (수학/영어 분리)
[갭 11] 가중택 대학 → Row 59 총점 사용
[갭 13] 홍익대 설정 완전성 ("영" → "국수영탐")
[갭 14] 인하대 변형 선택 최적화
```

### 2.3 제1원칙 정립

> **"0이면 다른 곳에 정당한 값이 있다"**

이 원칙을 문서화하여 향후 유사 문제 해결의 가이드라인으로 확립.

## 6. 종합 비평

### 6.1 잘한 점

1. **체계적 문제 해결**: 8개 갭을 순차적으로 발견하고 해결하는 과정이 논리적
2. **원칙 정립**: "0이면 다른 곳에 있다" 원칙은 향후 유사 문제에 재사용 가능
3. **버전 관리**: v1→v6.1 진화 과정이 명확하게 추적 가능
4. **문서화**: 의사결정 과정을 문서로 남겨 지식 자산화

### 6.2 개선 필요점

1. **테스트 데이터 한계**: 수학/탐구 미입력 상태로 전체 검증 불가
2. **하드코딩**: 대학명 매핑, Row 번호 등이 코드에 직접 삽입
3. **확장성**: 현재 21개 대학만 테스트, 전체 대학 검증 필요
4. **사용자 인터페이스 부재**: CLI 기반으로 일반 사용자 접근 어려움

### 6.3 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Excel 구조 변경 | Row 매핑 무효화 | 동적 Row 탐색 로직 필요 |
| 대학별 전형 변경 | 계산 오류 | 연도별 설정 분리 |
| 데이터 품질 | 잘못된 순위 | 교차 검증 로직 추가 |

### 6.4 총평

> **현재 단계는 "핵심 엔진 완성 직전"이다.**

Score Calculator의 핵심 로직(가중택, 변형 선택, Row 구조)이 완성되었으나,
실제 데이터로 전체 검증하고 사용자 인터페이스를 제공하는 단계가 남아있다.

**60% 완료 시점에서 가장 어려운 문제(NULL/0 값 해석)를 해결한 것은 긍정적이며,**
**남은 40%는 확장과 마무리 작업으로 상대적으로 명확한 경로가 보인다.**
